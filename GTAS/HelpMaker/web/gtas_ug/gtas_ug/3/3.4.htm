3.4 List View Page

List view pages present a list of records of the same type. GridTalk has many types of records. Some are associated with your company’s documents, but the majority of the records in GridTalk are configuration information used to define the processes that GridTalk performs on your company’s documents, to define the partners your GridTalk communicates with, and the methods it uses for such communication. Regardless of the record type the list views in which the records appear all have much the same features.

Presented below for your inspection is a typical list view:

At the top of the page is a heading that indicates what types of records are contained in the list view. In this case “Mapping Files”. Each row in the table corresponds to a single record.

The names of the columns are shown in the table header, and these names correspond to fields in the record. The first column is labeled “Edit” and each row contains an image. This image is a shortcut that you may click to open the record for editing. When you open a record for editing, its “detail view” will usually appear in a new browser window. . (The use of detail views will be explained later).

The first column after the Edit column contains an image and some text that is underlined. The contents of this column are also a link, in this case to allow you to open the record for viewing in a new detail view window.

For some records, the contents of other columns may also be underlined. There are some cases in which this occurs. The first is when the field of the record to which that column corresponds contains a reference to another record, in which case the link will open a detail view of the record referred to in a new window. The second case is where the field contains a file. In this case clicking the link will allow you to either open the file in a new browser window for viewing or download the file to your computer – this depends on the type of the file and the way in which your web browser has been configured.

Above the table are three links. These links (Refresh, Delete, and Create) are standard for most list view pages. Some list view pages have more links where there is a specialized functionality, while some do not show all three of the links.

The first link, “Refresh” is used to refresh the list view with the most current information. If for example another user edits one of the records, its details will not be updated until you refresh the list view or go to a different page and return . For some record types, such as Documents new records might be coming in from an external source, and clicking Refresh will update the list view to show any new records. (This Refresh link works differently to the refresh button provided by your browser. As mentioned earlier in the documentation you should avoid using the browsers refresh button in GridTalk due to the dynamic nature of the GridTalk application. If the documentation asks you to refresh a list view, it means you should use the Refresh link that is provided on the list view page).

The second link provided is Delete . The delete link allows you to remove records from GridTalk. The type of record to be deleted is also indicated by the text of this link. In the screenshot presented it reads “Delete Mapping Files”. Not all records may be erased, and for some list views, the delete link might not be present. To delete a record, or a set of records, they need to be selected. You will note that the leftmost column of the table contains small boxes, with one for each row. These are known as “Checkboxes”. When you click on a checkbox, a tick will appear in it. Clicking a checkbox that has a tick will remove the tick. The checkboxes in the list view page allow you to select a record or group of records. In addition to the checkbox being checked the row in the table will change color. To delete a group of records, you select those records that you wish to delete and having done so click Delete , at which point you are presented with what is called a “dialog box” which asks you to confirm that you really wish to delete the specified records. If you did not intend to click delete or you realize that you have selected an incorrect record, you may back out at this point by clicking the “Cancel” button. Clicking “Ok” will cause the delete operation to proceed.

The screenshot below shows some “Partner Type” records being selected for deletion:

You have probably noticed that in addition to the checkboxes in each individual row of the table, there is also one in the header of the table to the left of the column names. This checkbox allows you to select all the rows on the current page of the list view (we will explain more about list view paging shortly). If no records or some but not all records are selected, then clicking the select-all checkbox will cause all records to be selected. However, if all the records are already selected, then clicking the select-all checkbox will cause all to be unselected.

As mentioned above, not all records can be deleted. If, for some reason, a record cannot be deleted and you try to delete it, GridTalk will report this to you in an error message. (Any records in the group you selected that could be deleted will have been deleted by the time you see this message telling you about that ones that could not be erased). (In this screenshot, the “Business Entity” record could not be deleted, as it is a record that is needed for the correct operation of GridTalk. Certain configuration records of various types are also preconfigured into GridTalk and may not be deleted).

The other reason why we are not allowed to delete a record is that otherwise the GridTalk configuration would not be in a consistent state.

Below is presented an example screenshot that illustrates the error shown when you try to delete a record that has other records referring to it. We call such records “dependencies” as they “depend” on another record.

If there is more than one record that cannot be deleted from a group that you select, then each will be indicated individually in the error message. Where there are dependencies the types of dependant record, and the descriptions of each individual dependent record are also provided.

One point to note about the error messages is the little black triangles. These triangles may be clicked to hide or reveal details in much the same manner as the plus symbols in the navigation tree may be clicked to hide or reveal child nodes. When the error is presented to you, the triangle may be pointing to the right, in which case you will need to click it to reveal the hidden “section” containing further details.

The third link in the trio of links located above the table is Create . Like the delete link, it also specifies the type of record to be created. When you click on it, another browser window will open containing a “detail view” of a new instance of that record type.

The final elements of interest in the list view are those that relate to paging. To cut down on network traffic and server workload, the list views that have a lot of records are broken up into a number of pages. The paging tools are found below the table of records. The current page and the number of pages are shown, as the range of records this relates to. The number of records that show on a page is usually 10, however your GridTalk administrator may have configured a different number.

To the right of the text displaying the current page information are some arrows, and between the two sets of arrows are some page numbers. These page numbers are all links, with the exception of the one relating to the currently displayed page in the list, which is grayed out. As would be expected, clicking the number will take you to that page. Of the arrows the outermost will take you to the first and last pages, while the inner arrows will take you to the previous and next pages respectively. If you are on the first or last page, two of the arrows will not be clickable and will be grayed out.
