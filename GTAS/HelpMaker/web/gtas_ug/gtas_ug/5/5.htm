5 Partner

As a user, it is unlikely that it will be useful to define the recipients of the documents that you want to send out simply by their Business entity ID. It is more appropriate to let the user map the recipients to identifiers already known to the company. For example, identifier used in the backend ERP system.

Each partner is defined by a number of parameters:

Partner ID

This may be a number or a combination of characters and symbols. If you have a back-end application such as an ERP system, the partner IDs may already have been generated by this application.

For example: 0000001000

Using an ID that has been pre-generated by a back-end application will help GridTalk to automate the sending process for documents created by this application.

Partner Name

This is the name of the enterprise or even department that you exchange documents with.

For example: Compass Inc.

Partner Type

This is a classification of the broad type to which the partner belongs, based on its dealings with you. The GridTalk administrator should define the partner types.

For example: Supplier

Partner Group

This is a classification of the broad group to which the partner belongs, based on its dealings with you. The GridTalk administrator should define the partner groups.

For example: Local Suppliers
