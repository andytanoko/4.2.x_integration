14.1 Searching Archived Process Instances

Step 1. Click the Archive Search tab. This will bring you to the Archived Process Instance Search page by default.

Step 2. Enter the search criteria as needed:

Process Definition

This field indicates the type of the archived process instances to search for.

State

This field indicates the state of the archived process instances to search for.

Partner ID

This field indicates the Partner ID of the partner that was involved in the archived process instances to search for.

Partner Name

This field indicates the name of the partner(s) that was involved in the archived process instances to search for. The application would perform a wild card search based on the value input in this field.

From Start Time (YYYY-MM-DD) (HH:MM)

This indicates the starting date and time range to match the start times of the archived process instances to search for. The date portion can be entered without the time portion, which is then assumed to be 00:00. The time portion is automatically attached with the second and milliseconds portion 00.000. The matching archived process instances will have start times later than or same as the specified date and time.

To Start Time (YYYY-MM-DD) (HH:MM)

This indicates the ending date and time range to match the start times of the archived process instances to search for. The date portion can be entered without the time portion, which is then assumed to be 23:59. The time portion is automatically attached with seconds and milliseconds portion 59.999. The matching archived process instances will have start times earlier than or same as the specified date and time.

Doc. No.

This indicates the document number of the request document of the archived process instances to search for. The application would perform a wild card search based on the specified value for this field.

User Tracking ID

This indicates the user tracking ID of the request document of the archived process instances to search for. This is only possible if the user tracking identifier XPath was specified in the Process Definition when the process instances were running.

From Doc. Date (YYYY-MM-DD)

This indicates the starting date range to match the document date of the request documents of the archived process instances to search for. The specified date is automatically attached with the time portion 00:00:00.0. The matching archived process instances would have its initial request document with document date later than or equal to this specified date.

To Doc. Date (YYYY-MM-DD)

This indicates the ending date range to match the document date of the request documents of the archived process instances to search for. The specified date is automatically attached with the time portion 23:59:59.999. The matching archived process instances would have its initial request document with document date earlier than or equal to this specified date.

Remark

This indicates to search for archived process instances marked as Valid or Invalid during archival. A Valid archived process instance is termed as such if, during archival, all expected related records and files exist and are archived as a whole as one process instance. An Invalid archived process instance is termed as such if, during archival, any expected related records or files could not be found and not archived with the process instance. In short, archived process instances marked Invalid are those that have missing document records or files during the archival.

At least one of the above criteria must be specified in order to perform the search.

Doc. No. , From Doc. Date , and To Doc. Date criteria only work if the respective information is extracted into the GridDocument using XPath during folder transition or header mapping rules. See Chapter 8.2.8.

Step 3. Click Search to submit the entered search criteria.

Step 4. When the search is done, the Archived Process Instance Search Result page will be displayed, showing the submitted search criteria and the lists of the archived process instances that matched the search criteria.

Note that, for archived process instances marked Invalid, the Errors column would display the errors that were encountered when the process instances were archived. Otherwise, the Errors column would be empty. The errors are one of the following two values:

· No Docs – this means that no documents have been found for the process instance.

· Docs invalid – this means that some documents are missing or some files of the documents are missing.

Step 5. To sort the order of the presented list, click on the column header of the process instance field you want to sort by.

Step 6. To view the details of a process instance, click on hyperlink in the Process Instance ID column of the row you want to view. This will pop up a window displaying the Archive Process Instance detail page showing the process instance in more details and the documents that are associated with that process instance. You may view the details of multiple process instances at the same time; different pop-up windows are opened for each process instance.

The upper section of the page shows the general information of the archived process instance, and the bottom section shows the list of the documents that are associated with that process instance.

Process Instance ID

This indicates the process instance id of the process instance.

Doc. No.

This indicates the document number of the initial request document associated with this process instance. This value will only be present if document number is present in that document.

Partner ID

This indicates the partner id of the partner that is involved in that process instance.

State

This indicates the state of the process instance when it was archived.

Start Time

This indicates the start date/time of the process instance.

End Time

This indicates the end date/time of the process instance. This value will only be present if the process instance has completed, either successfully or abnormally, at the time of the archival.

User Tracking ID

This indicates the user tracking id of the initial request document associated with that process instance. This value will only be present if user tracking id is present in that document.

Doc. Date

This indicates the document date of the initial request document associated with that process instance. This value will only be present if document date is present in that document.

Retry Number

This indicates the number of retries that application had attempted to resend the request document of that process instance.

Failed Reason

This indicates a short description on the reason why the process instance had failed. This value will only be present if the process instance was in one of the abnormal states at the time of archival.

Detail Reason

This indicates the detail reason why the process instance had failed. This value will only be present if the process instance was in one of the abnormal states at the time of archival.

Errors

This indicates the errors that were encountered when the process instance was archived. This value will only be present if some related documents and files are missing.

Step 7. To sort the order of the associated documents, click on the column header of the field of the GridDocument you want to sort by.

Step 8. To view the details of a document, click on the hyperlink in the GridDoc. ID column of the document you want to view. This will display the Archived Document detail page. See Chapter 14.3.

Step 9. You may close the Archived Process Instance detail page by clicking on the window close button.

Step 10. To submit another search, click the browser Back button to return to the Process Instance Search page, or click the Process Instance Search link on the left navigation pane. Repeat from Step 2 onwards.
