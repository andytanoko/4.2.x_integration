14.2 Searching Archived Documents

Step 1. Click the Archive Search tab. This will bring you to the Archived Process Instance Search page by default.

Step 2. Click Document Search on the navigation pane. This will bring you to the Archived Document Search Page.

Step 3. Enter the search criteria as needed:

Partner ID

This indicates the partner ID of the partner that was involved in transactions of the archived documents to search for.

Partner Name

This indicates the name of the partner that was involved in the transactions of the archived documents to search for. The application would perform a wild card search based on the value entered.

Folder

This indicates the folder of the archived documents to search for.

Doc. Type

This indicates the document type of the archived documents to search for.

User Tracking ID

This indicates the user tracking ID of the archived documents to search for.

From Create Date (YYYY-MM-DD) (HH:MM)

This indicates the starting date and time range to match the create date of the archived documents to search for.

To Create Date (YYYY-MM-DD) (HH:MM)

This indicates the ending date and time range to match the create date of the archived documents to search for.

Create Date refers to the date and time that the document is being accepted and saved into the application.

From Sent Date (YYYY-MM-DD) (HH:MM)

This indicates the starting date and time range to match the sent date of the archived documents to search for.

To Sent Date (YYYY-MM-DD) (HH:MM)

This indicates the ending date and time range to match the sent date of the archived documents to search for.

Sent Date refers to the date and time that the document is being sent out from the sender’s end.

From Received Date (YYYY-MM-DD) (HH:MM)

This indicates the starting date and time range to match the received date of the archived documents to search for.

To Received Date (YYYY-MM-DD) (HH:MM)

This indicates the ending date and time range to match the received date of the archived documents to search for.

Received Date refers to the date and time that the document is being received at the recipient’s end.

Doc. No.

This indicates the document number of the archived documents to search for. The application would perform a wild card search based on the specified value for this field.

From Doc. Date (YYYY-MM-DD) (HH:MM)

This indicates the starting date and time range to match the document date of the archival documents to search for.

To Doc. Date (YYYY-MM-DD) (HH:MM)

This indicates the ending date and time range to match the document date of the archived documents to search for.

Remark

This indicates to search for archived documents marked as Valid or Invalid during archival. A Valid archived document is termed as such if, during archival, all expected related records and files exist and are archived as a whole as one document. An Invalid archived document is termed as such if, during archival, any expected related records or files could not be found and not archived with the document. In short, archived documents marked Invalid are those that have missing records or files during the archival.

At least one of the above criteria must be specified in order to perform the search.

For From Create Date , To Sent Date , To Received Dat e, and To Doc. Date , the date portion can be specified without the time portion, which is then assumed to be 00:00. The time portion is automatically attached with the seconds and milliseconds portion 00.000. The matching archived documents would have its create/sent/received/doc date later than or equal to the specified date and time.

For To Create Date , To Sent Date , To Received Date , and To Doc. Date , the date portion can be specified without the time portion, which is then assumed to be 23:59. The time portion is automatically attached with the seconds and milliseconds portion 59.999. The matching archived documents would have its create/sent/received/doc date earlier than or equal to this specified date and time.

Doc. No. , From Doc. Date , and To Doc. Date criteria only work if the respective information is extracted into the GridDocument using XPath during folder transition or header mapping rules. See Chapter 8.2.8.

Step 4. Click Search to submit the entered search criteria.

Step 5. When the search is done, the Archived Document Search Result page will be displayed, showing the submitted search criteria and the list of archived documents that matched the search criteria.

Note that, for archived documents marked Invalid, the Errors column would display the errors that were encountered when the documents were archived. Otherwise, the Errors column would be empty. The errors are a combination of the following values:

· Missing Audit – this means that the audit file of the document should exist but could not be found

· Missing Receipt Audit – this means that the receipt audit file of the document should exist but could not be found

· Missing User doc – this means that the user document file could not be found

· Missing Gdoc – this means that the GridDocument header file could not be found

· Missing Attachment – this means that there should be attachment(s) to the document but could not be found.

Step 6. To sort the order of the present list, click on the column header of the document field that you want to sort by.

Step 7. To view the details of an archived document, click on the hyperlink in the GridDoc. ID column of the document you want to view. This will display the Archived Document detail page. See Chapter 14.3.

Step 8. To submit another search, click the browser Back button to return to the Document Search page, or click on the Document Search link on the left navigation pane. Repeat from Step 3 onwards.
