16.2.1 To enable AS2 messaging with a partner

Step 1. Import one or more partner certificates into your GridTalk. Please note that the partner may use an encryption certificate that is different from the signature certificate. In this case, both certificates need to be imported.

Step 2. Next configure Partner channel. To do this, you need to configure packaging, security and communication profiles. Follow the steps A, B and C.

Step A:

For configuring packaging profile, choose AS2 for the packaging type.

When AS2 is selected, the first extension field, that is, Is Acknowledgement Required checkbox is displayed by default.

Configure the other extension fields accordingly to your specific requirement.

Step B:

For configuring security profile, if any security action (encryption, signature or compression) is intended, choose “S/MIME” for each of the intended actions. Then specify the corresponding parameters such as encryption algorithm, signature algorithm, etc. Refer to Chapter 7.3 for more details on encryption configurations.

But if more than one security action is selected, specify the action sequence using the dropdown list next to the Sequence field. The Sequence options are self-explained.

Step C:

For configuring communication profile, simply select and key in the HTTP URL.

After you have done with the above steps, create a new channel that refers to the profiles you have just created. If you want to create more channels that are needed by this partner, repeat the same steps as above.

Step 3. Configure Partner business entity. Assign all channels in Step 2 to this Business Entity. Click “domain identifier” tab, choose “AS2 identifiers” and supply a valid AS2 identifier (Take note that it is advised to use only English letters and numbers and to avoid special characters).

Step 4. Configure Partner. Among the list of channels, choose one as the default channel. This channel will be used if you do not configure a trigger for a particular document type and for this partner, or you do not specify which channel to use in that trigger configuration. If only AS2 is applicable to this partner, choose an AS2 channel as the default Channel.

Step 5. Proceed to configure Import Trigger. By doing so, you need to specify the retry count and retry interval as well as to specify which channel to use in case the partner has multiple channels.

For a typical configuration, just create a level 4 trigger for a particular document type and the intended partner, specify the retry count and retry interval, and among the list of channels that belong to this partner, select the channel that corresponds to the chosen document type. For example, the channel that encrypts the message has to be chosen for the document type such as “AS2_Encrypted_Message”

Step 6. Create certificate mapping. Choose the partner. Select partner’s signature certificate for Partner Certificate and your encryption certificate for Own Certificate.

Step 7. If you need to send AS2 message over SSL to the partner, import the root certificate of the partner SSL certificate into GridTalk Truststore. Refer to Chapter 7.5 for more details.
