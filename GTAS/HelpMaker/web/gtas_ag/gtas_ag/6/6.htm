6 Alert Notification Centre

Alerts are used to notify a user of what is going on in GridTalk. An alert is triggered when a certain event occurs at the GridTalk server.

Every alert is of a certain predefined type, which can be broadly categorized as follows:

  • License alerts – these are used to signal license expiring/expired.
  • Document event alerts – these are used to signal receive/export/receive acknowledgement.
  • Document tracking alerts – these are used to signal reminders/receive response. Processing error alerts – these are used to signal failure notification/receive exception/send/export/import/receive errors.
  • Application alerts – these are used to signal administrative states. (Take note that these alerts are not yet supported in this version).
  • User alerts – these are used to signal document mapping failure/user procedure failure/user-defined notifications.
