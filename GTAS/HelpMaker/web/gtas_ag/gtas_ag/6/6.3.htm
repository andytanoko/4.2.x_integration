6.3 Configuring Message Templates

Whenever an alert is triggered, a corresponding action will be taken. This action in turn depends upon the message Template it is associated. Based on the message template, the alert will be triggered to e-mail or log.

The message template is used to define a layout of the alert message to be shown to the user, when the alert occurs. It may embed some run-time values from some data objects using Object Field Reference Tags (OFRTs). The message template is used by the system to generate the message to be sent as an alert notification. Customization of alert message text is accomplished in the message template files through the use of Object Field Reference Tags.

GridTalk provides the list of default email and log message templates which you can use to send the alert messages to the user. See Appendix on what these message templates are configured for.

There are three forms in which an alert message can be presented to users:

  • Email – the alert message will be sent as an email notification to a number of recipients
  • Log – the alert message will be logged to a file
  • JMS – the alert message will be sent in a form of JMS message to targeted JMS destination
