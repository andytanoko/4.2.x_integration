The following explains what each configuration A-G means:

Configuration

Meaning

A

An Alert Trigger must be configured and enabled at Level 0 for the respective Alert Type.

B

An Alert Trigger must be configured and enabled at one of the levels (0-4) for the respective Alert Type. For all cases, the level at which the Alert is triggered is based on a criteria evaluation, starting from Level 4 down to Level 0.

C

A “Raise Alert” Workflow Activity must be configured in a Partner Function for the respective Alert Type. When the Partner Function executes the “Raise Alert” Workflow Activity, an Alert will be triggered at the level based on the criteria evaluation as in B.

D

A “Raise Alert” Workflow Activity must be configured in a Partner Function for “User Defined” Alert Type. The user defined Alert specified in the workflow activity will be raised.

E

A User Procedure must be configured with return value evaluation to trigger an Alert of the “User Defined” Alert Type when it matches a condition. When the User Procedure executes and returns a value, the return value is evaluated. If it matches the condition, the specified Alert will be raised.

F

A “Raise Alert” Workflow Activity must be configured in a Partner Function for the “Document Response Received” Alert Type. A Response Track Record must be configured with “Response Alert to raise”. When the Partner Function executes the “Raise Alert” Workflow Activity, if the document in Partner Function is a response document for a sent document under response tracking, the specified alert will be raised.

G

A Reminder Alert must be configured in a Response Track Record. When the Days to Reminder have passed and no response document is received for a sent document under document tracking, the specified alert will be raised.
