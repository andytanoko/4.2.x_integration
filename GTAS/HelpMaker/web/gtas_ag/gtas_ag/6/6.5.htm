6.5 Configuring Response Track Records

A Response Track record is used to activate document response tracking when the user decides to receive the partner's response for the sent document.

For example, when defining a Response Track Record, you have to state the Sent Doc Type as 3A4P0Request and Response Doc Type as 3A4POConfirmation.

Therefore, the main purpose of creating a Response Track Record is to raise an alert on two occasions, that is, when the response document is received or when the response document is not received within some specified number of days.

There are two ways of raising an alert:

  • Select the alert in the Receive Response Alert field when the document is received.
  • Define Reminder alerts when the response document is not received within a specified number of days.

The Document Identifier and Start Track Date are very important parameters to ensure the working response track record. Take note that both the Sent Document and Response Document Identifier must match with each other otherwise the tracking process will not work.
