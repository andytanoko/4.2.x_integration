6.7 Configuring Email Properties

In order for email alert notifications to be delivered, the SMTP server must be provided. The Email Configuration panel allows you to configure the SMTP server host and port, and, optionally, SMTP server authentication user and password.

Emails may fail delivery due to temporary downtime of SMTP server or network. In such situations, GridTalk can take care of re-delivering the emails when the network or SMTP server connection is restored. You may configure the maximum number of retries to re-deliver the email notifications and the interval between each retry. In addition, you can also choose to save a copy of each email that failed to re-delivery after maximum retries.

Step 1. Click the Email Configuration . This will bring you to the Email Configuration view page.

Step 2. Click on the Edit Email Configuration link. This will bring you the Email Configuration page.

Enter the following fields for the Email Configuration:

Smtp Server Host :

This field allows you to specify SMTP server host.

Smtp Server Port :

This field allows you to specify SMTP server port.

User Name :

This field allows you to specify the username to connect to the SMTP server.

Password :

This field allows you to specify the password to connect to the SMTP server.

Retry Interval :

This field requires you to specify the time interval in seconds to retry sending the failed mails.

This value should be at least 180 seconds.

Max Retries :

This field requires you to specify the maximum number of times it should retry sending the failed mails. To disable retries this field should be set to –1.

Save Failed Emails :

This field allows you to save the failed mails if you choose Yes.

Step 3. To confirm the setting, click Apply.
