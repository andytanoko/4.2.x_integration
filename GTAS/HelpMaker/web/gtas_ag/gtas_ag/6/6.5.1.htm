6.5.1 To create a new response track record

Step 1. Click Response Track Record. The Response Track Records list view page with the following columns in the table is displayed.

Edit

This field allows you to update the properties of the document type.

Sent Document Type

This field indicates the type of document that you have sent to your partner.

Start Track Date XPath

This field indicates the start date of the record.

Response Document Type

This field indicates the type of document, which you have received or not received from your partner

Step 2. To create a new response track record, click Create Response Track Record. The Response Track Record detail view page is displayed.

Step 3. In the Sent Document pane, select Sent Document Type from the list box. For example, 3A4PORequest. If the document type is not listed here, click on the icon to create a document type.

Step 4. Enter Sent Document Identifier XPath. The user needs to specify the XPath in the requesting document for extracting the document identifier. If not specified, the UDocNum in the GridDoc will be used instead.

Step 5. Enter Track Date XPath. The user needs to specify the XPath in the requesting document for extracting a date from which the tracking will start. If not specified, the DT Sent Start in GridDoc will be used. For example, if the start date is 24th Feb 03 in the sent doc, and you define a Reminder Alert with 2 days to reminder, then on the 26th Feb, if the response document is still not received, a document reminder alert will be raised.

Step 6. In the Response Document pane, select Response Document Type from the list box. For example, 3A4POConfirmation. If the document type is not listed here, click on the icon to create a document type.

Step 7. Enter Response Document Identifier XPath. The user needs to specify the XPath in the response document to extract document identifier to match with the sent document identifier.

Step 8. Next proceed to the Reminder Alerts pane. The columns in the Reminder Alerts section are:

Edit

This field allows you to update the properties of the selected day.

Days to Reminder

This field indicates the number of days after the start tracking date when the response document is still not received to raise document reminder alerts.

Alert to raise

The field allows you to configure the document reminder alerts.

Document Recipient XPath

This field is the recipient of the sent document. The XPath will be used to extract an email code from the sent document. The extracted email code will be used for lookup in a emailcode-ref.xml file (under <GRIDTALK_HOME>/data/GNapps/gtas/data/alert/email) for email addresses of the recipient of the reminder alert. See Chapter 6.5.2The Email Code Reference File.

Document Sender XPath

This field is the sender of the sent document. The XPath will be used to extract an email code from the sent document. The extracted email codes will be used for lookup in the emailcode-ref.xml file (under <GRIDTALK_HOME>/j<boss>/bin/gtas/data/alert/email) for recipients of the reminder alert. See Chapter 6.5.2The Email Code Reference File.

Step 9. Next go to the Receive Document pane.

Step 10. Select Receive Response Alert from the list box. If the alert you want is not listed here, click on the icon to create a new alert.

Enter Alert Recipient XPath. The user needs to specify the XPath in the response document for extracting email code. The email code will be used for lookup in the emailcode -ref.xml file for the recipients of the response alert.

Step 11. To save the creation, click Save Response Track Response. This record will appear in the Response Track Records list view page.
