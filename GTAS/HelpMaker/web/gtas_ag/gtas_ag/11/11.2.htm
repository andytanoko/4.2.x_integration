11.2 Backend Web Services Integration

As discussed earlier in the Overview of Backend Integration, GridTalk integrates this backend with the web service to allow any web service clients to import documents into the applications (e.g. XB, SAP) via web services.

As web services are language neutral and platform independent, there can be many ways of invoking the services. Have a look at the actual backend web service wsdl file as shown below. This is done by pointing your URL to the following:

http://<host>:<port>/gridtalk/webservice/services/BackendImport?wsdl

where the host refers to the IP address of the machine hosting GridTalk.

There are six variants of the backend import services:

  1. backendImport1 : backend import without attachments, using DataHandler to transfer file
  2. backendImport2 : backend import with attachments, using DataHandler to transfer files
  3. backendImport3 : process-oriented backend import with attachments, using Data Handler to transfer files
  4. backendImport4 : backend import without attachments, transferring files as byte contents
  5. backendImport5 : backend import with attachments, transferring transaction doc as byte contents and transferring attachment using DataHandler
  6. backendImport6 : process oriented backend import with attachments, transferring transaction doc as byte contents and transferring attachment using DataHandler
