11.4.2 To assign the web service to the partner

The Service Assignments functionality is used for managing web services that each partner is allowed to access or invoke.

Step 1. Go to Web Services tab. The Own Services list view page is displayed by default.

Step 2. To configure and assign the partner access to your web services, click Service Assignments under the Web Services menu. Then the Service Assignments list view page detail is displayed.

Step 3. Click Create Service Assignment to bring up the Service Assignment detail view page.

Step 4. Enter the following fields to configure the service assignment:

User Name

Select the partner you want to assign the web services so that the partner is allowed to access or invoke the web service. Click to create the partner if the partner you authorize is not listed in the drop-down listbox.

Password

Enter the authentication password for the partner. The password should contain at least six characters.

Take note that the user name and password are used for HTTP basic authentication when the partner attempts to invoke you web services.

WebServices

Select the list of available services that you allow the partner to access. These services are already created earlier. Click to create the web service if the web service to assign is not listed in the list box.

Step 5. Once you have finished input for all the fields, click Save Service Assignment. The new service assignment will be added to the Service Assignments list view page. To modify the details of the Service Assignment, just click on the Edit icon for the Service Assignment record in the list view page. To delete the service assignment, tick the checkbox against the record and click Delete Service Assignment.
