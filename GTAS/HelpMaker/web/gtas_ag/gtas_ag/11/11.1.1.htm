11.1.1 To create a new SOAP-enabled user procedure

Step 1. Click User Procedure.

Step 2. To create a new user procedure, click Create User Procedure.

Step 3. In the User Procedure detail view page, key in WSDBUP and Dash Board Web Service User Procedure for the Name field and the Description field respectively.

Step 4. Select Web Service for the Procedure Type. Then you have to create new parameters for that type by clicking Edit Web Service Configuration. When the user chooses Edit Web Service Configuration, GridTalk auto detects and list all available methods (services) in the WSDL file so that the user does not have to know beforehand what is the method and parameters to use. They can use the parameters information here to help them configure the Procedure Parameter List.

Step 5. Select Method Name by clicking its radio button. This field represents one of the methods found in the Class Name.

Step 6. Click Save Web Service Configuration to save the configuration and to return you to the User Procedure page.

Step 7. Proceed to define Return Data Type. This field is the data type of the return value after executing the jar file. Select Integer.

Step 8. Select Alerts. See Chapter 6 for creating Web Service alerts.

Step 9. Next go to the Procedure Parameter List pane to configure several parameters for that Web Service user procedure you have created.

Step 10. Then proceed to the Procedure Return List pane to create a new procedure return condition. For alerts, see Alert, Action and Template Message in the Chapter 6.

Step 11. Click Create Procedure Return Condition. The Procedure Return Conditions page is displayed.

Step 12. You may configure return conditions for “true”, which means successful invocation of the web service method, and/or for “false”, which means the invocation is unsuccessful.

Step 13. Click Save Procedure Return Conditions.

Step 14. Click Save User Procedure. Then the User Procedure to invoke a Web Service is successfully created.
