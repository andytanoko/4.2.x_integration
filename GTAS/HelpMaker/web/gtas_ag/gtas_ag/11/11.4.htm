11.4 Using Web Service Gateway

GridTalk provides the web service gateway as a means for authenticated partners and authorize partners to invoke enterprise’s internal web services.

Using this gateway, you will be able to manage a registry of web services and partner authorizations by:

· Configuring a service name and the WSDL and EndPoint URLs for each web service.

· Assigning authentication password (min. 6 characters) to each Partner and

· Authorizing the web services for invocation by each Partner
