11.3 Front Web Services Integration

As discussed earlier in the Overview of Frontend Integration, GridTalk integrates web service to allow any web service clients to deposit documents into the applications as well as the retrieving of document via web services.

As web services are language neutral and platform independent, there can be many ways of invoking the services. Have a look at the actual front end web service wsdl file as shown below. This is done by pointing your URL to the following:

Web service for depositing doc

http://<host>:<port>/gridtalk/webservice/services/ SoapMessageReceiverService?wsdl

Web service for retrieving doc

http://<host>:<port>/gridtalk/webservice/services/MessageRetrieveService?wsdl

where the host refers to the IP address of the machine hosting GridTalk.
