11 Web Services

In order for a web service to meet the needs of the enterprise, it needs to support the description of services using WSDL (Web Service Description Language) and the SOAP (Simple Object Access Protocol) transmission protocol.

Incorporated with the Web Service feature, GridTalk is designed to invoke and make a web service accessible over the Internet by using web-enabling technologies such as:

  • SOAP-Enabling User Procedure

This makes use of the executing engine to support invocation of web services via User Procedure.

  • Backend SOAP Interface

This allows invocation of backend import services via Web Service.

  • Front end SOAP Interface

This allows the retrieving of document and depositing of document via Web Service.
