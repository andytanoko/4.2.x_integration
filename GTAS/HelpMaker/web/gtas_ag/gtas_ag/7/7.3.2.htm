7.3.2 To create a security profile for a Partner

Step 1. Click Communication.

Step 2. Click Partner under Security Profiles.

Step 3. In the Partner Security Profiles list view page, click Create Security Profile.

Step 4. In the Partner Security Profile detail view page, there are additional fields such as Signature Type and Compression Type. Enter the necessary fields. Some fields that are highlighted blue are mandatory. For example, Name, Description etc.

Name:

This field gives the name for the security profile.

Description:

This field gives the descriptive purpose for that security profile.

Partner:

This field specifies if that security profile is a partner or not. Ensure this is ticked.

Encryption:

This field gives three options to select. The options are None, S/MIME and Asymmetric (Public Key Cryptography). If you choose Asymmetric (Public Key Cryptography), the Encryption Level and Encryption Certificate will be enabled. For the S/MIME encryption chosen, the Encryption Level, Encryption Certificate and Encryption Algorithm will be enabled for you to select.

The Encryption Level determines how difficult your level is set so that an unauthorized user is unable to read the encrypted message. For Asymmetric (Public Key Cryptography), the different levels available are 1024 bits, 512 bits, 256 bits and 64 bits. For S/MIME, the different levels available are 128 bits, 168 bits, 40 bits and 64 bits.

The Encryption Certificate is used for encrypting messages. You should select an imported Partner Certificate.

The Encryption Algorithm is used to indicate the algorithm to use to encrypt messages. Supported algorithms are 3DES and RC2. Using 3DES algorithm requires the use of 168-bit encryption level only.

Signature:

This field gives two options to select. The options are None and S/MIME. If you choose S/MIME, the Digest Algorithm and Signature Certificate will be enabled. It provides authentication of the signer as well as the integrity and authentication of the signed data. It is a security standard that allows the verification of the identity of their partners involved in a business process transaction as well as to ensure and protect the integrity of the document data. Once the document data is signed, it can be secure and cannot be changed.

The Digest Algorithm determines how the users can sign the part of their document data to show others who they are. The different algorithms available are MD5 and SHA1. MD5 creates a 128-bit message digest to verify the integrity of data. SHA1 creates a 160-bit message digest that is more difficult than MD5 when it comes to verifying data integrity.

The Signature Certificate is used for signing the message digest. You should select own GridTalk Certificate or an imported Private Certificate.

Compression:

This field gives two options to select. The options are None and S/MIME. If you choose S/MIME, the Compression Method and Compression Level will be enabled. This is to ensure more space or save transmission time especially when you receive or send larger document files over the Internet to your partners

The Compression Method describes how document files are compressed so that these files have to be returned to their exact form when uncompressed. The only and available compression method is ZLIB (general purpose data compression / decompression library).

The Compression Level determines how much compression you want to use. The available compression levels are 0, 1, 2, 3, and 4. The compression level 4 is the highest file compression whereas zero is the lowest file compression.

Step 5. Click Save Security Profile. The newly created security profile will be displayed in the Partner Security Profiles, and All Security Profiles list view pages.

Step 6. To edit the Partner Security Profile, click on the Edit button . Update the entries as necessary. Click Save Security Profile.

Step 7. To delete the partner Security Profile, select the check box of the partner security profile you want to delete. Click Delete Security Profiles. The selected security profile will be removed from the All Security Profiles, and Partner Security Profiles list view pages.
