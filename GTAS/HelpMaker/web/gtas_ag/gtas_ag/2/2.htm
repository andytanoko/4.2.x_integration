2 Registration

In order to login to GridTalk, you will need to have a GridTalk user record created for you. Like other configuration records, these records are created from the web interface and so require you to be logged in already. To allow you to log in the first time there are two predefined accounts created.

Take note that you can login with the same user account on multiple machines.

If you are an administrator, just key in “admin” for User ID, followed by “admin” for Password. To confirm the entries, click Login. Take note that you can press the Enter key instead of clicking Login.

Step 1. Enter the user name. The default user name is “admin”

Step 2. Enter the password. The default password is “admin”

Step 3. If necessary, select the Timezone. This Timezone defaults to the timezone of the machine on which the browser is running. In most cases, this will be the same timezone as the user is in. However, in the case of remote users, this may differ. It is unfortunately not possible to automatically determine the user timezone as the browser only provides information on its offset from UTC, which does not include other necessary information such as daylight savings rules. The offset is used to narrow down the list of timezones. The available timezones are based on what is provided by the Java Virtual Machine in which the server runs and this often provides more than one timezone for the same place but with different names. The interface has been coded such that UTC time is always selectable. The selected timezone is used to format all date time information displayed so that it is in the local time. For example, if a document arrived at 9am Singapore time, a user in Singapore would see 9am displayed while the same document displayed to a user in Greenwich would display as 1am.

Step 4. Click on the Login button. The browser will show the Unregistered page with the Registration and Profile tabs. At the Registration tab, it shows the blank fields to be entered and, as you have not registered yourself yet, the information “Registration State Product not registered” is read here. Before being able to complete your registration, you must have the node-locked license file that will be provided by GridNode or its representatives. .

Step 5. Enter your GridNode ID

Step 6. Enter your GridNode Name

Step 7. To select the license file, click Browse. The license file contains your product key. You will receive the email with the attached file, which you can copy to any directory you want.

Step 8. Once done, proceed to click the Profile tab. The Company Profile panel displays all the blank fields to be entered. Take note that the Name, Language and Country fields are mandatory.

Step 9. Once done, click Register. Wait for a few moments until the browser takes you to the Register Product confirmation page.

This page requires you to enter and confirm your registration.

Step 10. Once done, click Confirm Registration to confirm your entries. Otherwise click Cancel Registration if a user has to repeat the registration. Wait for a few moments until you will see the Product Registration page with Registration State read as ‘Registered’. For updating your license, see Chapter 3.

Step 11. Once your product registration has been successfully completed, you are now in the User mode of the GridTalk application browser. To get into the Admin mode, click on the Admin hyperlink on the top right corner of the browser. You will see that the browser brings you to the Admin mode where Administration, Communication, Mapping, and Web Services tabs are presented in addition.

Take note that in the case you need to edit any of the mandatory fields, you can go to Registry to select Company Profile to make the changes or correct the entries.
