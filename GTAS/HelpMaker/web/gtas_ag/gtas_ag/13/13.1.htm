13.1 Archiving Documents and Processes

Step 1. Click the Administration tab. This will bring you to the Administration page.

Step 2. Click Archive Documents. Then the browser will bring you to the Archive Documents page where you can set the criteria to select the documents to be archived.

Step 3. Enter the following fields in the Archive Documents detail view page:

Name

This is an arbitrary name to identify the particular archival that you are performing right now.

Description

This is an optional field to give a much more detailed description of this archival.

Archive Type

This indicates the type of items to archive. Available options are: Document and Process Instance. Document archive type means that only matching GridDocuments and business documents are archived, regardless of whether they are participating in a process. Process Instance archive type means that matching Process Instances and also the participating GridDocuments, business documents and audit files are archived. It is recommended that a Process Instance archival be performed before a Document archival. This field is mandatory.

Enable Search

This indicates whether to enable the archived data for searching later on. Selecting this option would compress and store the archived data in an archive database. The archived documents are accessible from the GridTalk application using the Archive Search module. See Chapter 14.

Enable Restore

This indicates whether to enable the archived data for restoration later on. Selecting this option would compress and store the archived data in an archive folder. An Archive Summary file is attached to each archive, which is necessary for restoration. See Chapter 13.2 .

If both Enable Search and Enable Restore options are not selected, the archival is equivalent to as purging, i.e. permanently removing the documents and processes from GridTalk.

From Date (YYYY-MM-DD)

This mandatory field indicates the date portion of the starting date/time range to match based on the selected Archive Type.

From Time (HH:MM)

This mandatory field indicates the time portion of the starting date/time range to match based on the selected Archive Type.

To Date (YYYY-MM-DD)

This mandatory field indicates the date portion of the ending date/time range to match based on the selected Archive Type.

To Time (HH:MM)

This mandatory field indicates the time portion of the ending date/time range to match based on the selected Archive Type.

Partner

This optional field indicates to include the documents/process instances that involve the selected partners. For the partners that you want, select and click the arrow to move them to the right listbox. For those partners that you do not wish to match, select them in the right listbox and click the arrow to move them to the left listbox.

For Document archive type, the input date/time range will be matched against the document create date/time. For Process Instance archive type, the input date/time range will be matched against the process instance start date/time.

Step 4. If you select Document archive type, you will be prompted further for the folders and document types of the documents to match. For the folders and document types that you want, select and click the arrow to move them to the right listbox. For those items that you do not wish to match, select them in the right listbox and click the arrow to move them to the left listbox. At least one folder and document type must be selected in the respective right listboxes.

Step 5. If you select Process Instance archive type, you will be prompted with the process definitions to match and whether to include processes that are still in running state. For the process definitions that you want, select and click the arrow to move them to the right listbox. For those items that you do not wish to match, select them in the right listbox and click the arrow to move them to the left listbox. At least one Process Definition must be selected in the right listbox.

If you want to also archive process instances that are still in running state, select Yes for Include Incomplete Processes, otherwise leave it as No.

Step 6. After completing the necessary entries, click Generate Archive to start the archival process. The archival is performed as a background process and may take quite some time if there are a lot to archive based on the selected criteria. You may logout of the application while the archival is still in progress.

You will be notified by email upon the start and completion of the archival process, as well as upon failure of the archival process. The emails are sent to the Administrator user by default. To change the email recipients, edit the “ARCHIVE_START_EMAIL”, “ARCHIVE_COMPLETE_EMAIL” and “ARCHIVE_FAIL_EMAIL” message templates, following procedures in Chapter 6.3 . See Appendix Default Email Message Templates for the respective email templates.

The “ARCHIVE_COMPLETE_EMAIL” and “ARCHIVE_FAIL_EMAIL” emails will contain an attachment which is the archive summary file (an XML file) that indicates the result of the archival and the archival details. You may save this summary file for reference and also for restore later on if necessary.
