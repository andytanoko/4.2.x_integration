8.1.2 To create a new file type

GridTalk is able to support any file type for exchange and management, without the need for further configuration. However, configuring specific file types will allow GridTalk to identify a document’s file type by its filename extension and to view this document in the application which the Internet browser can support.

By default, GridTalk is already configured for the following file types and their corresponding viewing programs: 

File Type


Name

*.doc

Microsoft Word document

*.txt

Text document

*.xml

XML document

*.csv

CSV file

*.xsl

XSL file

GridTalk uses the default document viewer as defined in the system that the user uses to access the application.

Step 1. Click File Types under the Mapping folder. The File Types list view page is displayed where you see a list table of predefined file types.

Step 2. Click Create File Type. The File Type detail view page appears.

Step 3. Enter the mandatory fields such as File Type and Description.

Step 4. Click Save File Type. The new file type will be displayed in the File Types list table.

Step 5. To add more file types, repeat Step 2 through Step 4. If you want to view the file type, go to the File Types page where you will see the records listed in the table. Click on the selected file type’s hyperlink under the File Type column in the list table. If you wish to update the selected file type, click Edit File Type.

Step 6. To edit the existing or current file type, select the file type and then click on the corresponding Edit button . The File Type page is displayed. Update the entry of the file type as necessary. Click Save File Type.

Step 7. In the File Types page, tick the Select checkbox of the file type you want to delete. As you see a mark in the checkbox of the highlighted file type, click Delete File Type. Then you will be prompted with the message box asking, “Are you sure you wish to delete these File Types?” Click OK. The selected file type will be removed from the list table in the File Types page.
