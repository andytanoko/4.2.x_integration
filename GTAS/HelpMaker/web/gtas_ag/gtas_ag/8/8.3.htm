8.3 Configuring Mapping Rules

Mapping rules define how the contents of a document are managed when that document is copied, transformed, converted or split by GridTalk.

Transformation occurs when an XSL is used to transform an XML document either into another XML document or a text file.

Conversion occurs when the file type itself is changed. For example, from an Excel document to an XML document.

Split occurs when the user document is split into one or more XML documents. This is done using XPath to extract a list of unique values from the user document. Each unique value will produce a XML document that will only contain the record with that unique value.

Mapping rules in GridTalk may either be user or system defined. System mapping rules are defaults used by GridTalk in the absence of user mapping rules stating otherwise, to help track the movement of documents between the folders within GridTalk.

A mapping rule is allowed to contain one or more mapping files. There are two types of mapping rules in GridTalk: Header and Content.
