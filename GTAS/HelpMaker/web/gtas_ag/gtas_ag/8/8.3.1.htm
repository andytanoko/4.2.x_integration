8.3.1 To create a header mapping rule

A Header Mapping Rule is a XSL used for transforming the Grid Document. You are provided with five default header Mapping rules, which allow you to select to execute the mapping between the folders. You can create your own header Mapping rule to extract information from the user document/reference file into the Grid Document.

Step 1. Click Mapping Rules under the Mapping folder.

Step 2. Expand the Mapping Rules folder and click Header. The Header Mapping Rules list view page is displayed.

Step 3. Click Create Mapping Rule. The screen appears. Take note that the Header Transformation checkbox is ticked by default. Since it shows that the type of mapping rule is Header, the Type field is disabled.

Step 4. Specify the Name and Description for the Mapping Rule.

Step 5. Select the Mapping File from the list box. If the mapping file you want is not listed, click on the icon to create a new mapping file.

Step 6. Tick Transform With Source if you want to extract information from the user document (The user document has to be a XML file). The full path to the user document will be passed into the XSL mapping file through the parameter UDOC. You have to define this parameter UDOC in the XSL.

Step 7. Tick Transform With Ref Doc if you want to extract information from a reference file (Mapping File of type Reference Doc). Once this checkbox is ticked, you are prompted to enter Ref Doc by selecting from the list of defined reference document mapping file. If the reference document Mapping File is not listed, just click Create Mapping File to create it. The full path to the reference document will be passed into the XSL mapping file through the parameter REFDOC. You have to define this parameter REFDOC in the XSL.

Step 8. Click Save Mapping Rule. The new mapping file will be displayed in the Mapping Rules list view page.

Step 9. To add more Header mapping rules, repeat Step 3 through Step 8. If you want to view the mapping file, go to the Header Mapping Rules page where you will see the records listed in the table. Click on the selected mapping rule’s hyperlink under Name in the table. If you wish to update the selected mapping file, click Edit Mapping Rule.

Step 10. To edit the Header mapping rule, select it and then click on the corresponding Edit button . The Mapping Rule detail view page is displayed. Update the entries of the mapping rule as necessary. Click Save Mapping Rule.

Step 11. To delete the Header mapping rule, tick the Select checkbox of the mapping rule you want to delete. As you see a mark appear in that checkbox, click Delete Mapping Files. The message box, “Are you sure you wish to delete these Mapping Rules?” will appear. To confirm the deletion, click OK. The selected mapping file will be removed from the Mapping Rules and the Header Mapping Rules list view pages.
