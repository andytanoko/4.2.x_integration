8.2.9 To create a new Java binary mapping file

Java Binary is used to convert data files from one format to another. It is basically a set of Java classes that are created via the MapForce TM , hence it can handle a lot of different type of conversion MapForce is supporting. For now, the Java Binary can only take in one input file and output one single file. Once the Java Binary file is ready, the Java binary can be inserted into GridTalk.

Step 1. Click Mapping. Then click Java Binary under Mapping Files. The Java Binary Files list view page is displayed.

Step 2. Click Create Mapping File. The Mapping File detail view page is displayed.

Step 3. In the Mapping File detail view page, enter the mandatory fields.

Step 4. Select Java Binary for the Type field.

Step 5. Select the file by clicking Browse. The File selector is displayed to select the Java Binary file.

Step 6. Click Save Mapping File. The new mapping file will be displayed in the Mapping Files and Java Binary Mapping Files pages.

Step 7. To add more Java Binary mapping files, repeat Step 2 through Step 6. If you want to view the mapping file, go to the Java Binary Mapping Files page where you will see the records listed in the table. Click on the selected mapping file’s hyperlink under Name in the table. If you wish to update the selected mapping file, click Edit Mapping File.

Step 8. To edit the Java Binary mapping file, select the mapping file and then click on the corresponding Edit button . The Mapping File page is displayed. Update the entries of the mapping file as necessary. Click Save Mapping File.

Step 9. To delete the Java Binary mapping file, tick the Select checkbox of the mapping file you want to delete. As you see a mark appear in that checkbox, click Delete Mapping Files. Then you will be prompted to confirm the message box, “Are you sure you wish to delete these Mapping Files?” Click OK. The selected mapping file will be removed from the Mapping Files page and Java Binary Mapping Files page.
