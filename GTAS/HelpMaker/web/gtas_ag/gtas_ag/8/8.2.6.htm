8.2.6 To create schema mapping files in batch

Some new generation XML Schemas contains sub-schemas that are already well-defined by some other public standards or for re-use. The sub-schemas are themselves XML schema files which in turn may contain sub-schemas too. During XML schema validation, the XML schema and all sub-schemas must be present in order for the schema validation to perform successfully.

Since the usual way of creating Schema Mapping File only handles a single XML schema file, GridTalk provides a function to allow the users to create the Schema Mapping File as well as import the required sub-schema files into the application at the same time. This requires all XML Schema files to be contained within a single ZIP file.

Step 1. Click Mapping.

Step 2. Click Schema under Mapping Files. The Schema Mapping Files list view page is displayed.

Step 3. Click Batch import Schema Mapping Files. The Batch Import Schema Mapping Files detail view page is displayed.

Step 4. Click the Browse button to select the ZIP file containing all the XML Schema files to be imported. For example, the set of XML schema files for PIP4E1 Notify of Sales Report is contained in the ZIP file named PIP4E1_R11.00.00A_NotifyOfSalesReport.zip containing such contents as:

Step 5. After selecting the ZIP file, click Next. This brings you to the next screen to specify the XML Schema files to create as Schema Mapping Files entries.

Step 6. Specify the Base Directory, if you want the imported XML schema files to be located under additional common directory under the GridTalk schema directory (data/sys/validation/schema). Leave it blank if this is not necessary.

Step 7. Next, go to Schema Mapping section. For those XML Schema files that you want a Schema Mapping File record created, click Create Schema Mapping. This displays the Schema Mapping detail view page where the follow fields are mandatory to be entered:

Path

This presents a list of the folder paths found in the selected ZIP file. Select the path that the XML Schema to be created as Schema Mapping File record is located.

Filename

On selection of Path, this presents a list of the files found under the Path in the selected ZIP file. Select the filename of the XML Schema to be created as Schema Mapping File record.

Mapping Name

Enter the name of the Schema Mapping File to be created. This must not be the same as any existing Schema Mapping File record.

Description

Enter a description for the Schema Mapping File to be created.

Step 8. Click Save Schema Mapping to confirm the entries. This returns you to the Batch Import Schema Files page showing the Schema Mapping entry that you have just saved.

Step 9. To add more Schema Mapping entries, repeat Step 7 through Step 8. To modify information of a created Schema Mapping entry, click Edit button against the entry. To delete any created Schema Mapping entry, tick the checkbox of that entry, and click Delete Schema Mapping.

Step 10. To confirm all the entries, click Import Files to start imported the XML Schema files from the selected ZIP file. If the import is successful, all created Schema Mapping entries will be displayed as Schema Mapping Files in the Schema Mapping Files list view page.
