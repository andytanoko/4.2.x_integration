8.3.2 To create a content mapping rule

The Content Mapping Rule is used for manipulating the user document. When creating a content mapping rule, you need to specify the type that is used to split, transform or convert a user document.

Step 1. Click Mapping Rules under the Mapping folder.

Step 2. Expand the Mapping Rules folder, and click Content. The Content Mapping Rules list view page is displayed.

Step 3. Click Create Mapping Rule. The Mapping Rule detail view page appears. Take note that the Header Transformation checkbox is empty by default.

Step 4. Enter Name and Description.

Step 5. Next select Type. There are three available types such as Convert, Split, and Transform.

To use Split and Transform, your user document must be a XML file.

Step 6. Specify Source Doc Type by selecting the Document Type from the drop-down list. If the document type you want is not found, click on the icon to create a new document type. This indicates the document type for the input user document.

Step 7. Specify Target Doc Type by selecting the Document Type from the drop-down list. If the document type you want is not found, click on the icon to create a new document type. This indicates the document type for the output user document after the mapping rule is applied.

Step 8. Specify Source Doc File Type by selecting the File Type from the drop-down list. If the file type you want is not found, click on the icon to create a new file type. This indicates the file type of the input user document.

Step 9. Specify Target Doc File Type by selecting the File Type from the drop-down list. If the file type you want is not found, click on the icon to create a new file type. This indicates the file type of the output user document after the mapping rule is applied.

Step 10. Select the Mapping File from the list box. If the mapping file you want is not listed, click on the icon to create a new file type.

Step 11. If you have selected Split for Type, you need to define, additionally, the following fields:

XPath

This indicates a set of unique values in the user document that will be used for the splitting.

Param Name

This indicates the name of a parameter defined in the XSL that is used in the Mapping Rule for splitting. This parameter will contain each unique value extracted from the user document using the specified XPath.

Keep Original

If you want to keep the original XML document in the document folder, tick this checkbox.

Step 12. If you have selected Transform for Type, you need to define, additionally, the following fields:

Transform With Header

If you want to extract information from the Grid Document into your user document, tick this checkbox and define a parameter GDOC in the XSL.

Transform With Ref Doc

If you want to reference a Reference Doc in the XSL during transformation, tick this checkbox and define a parameter REFDOC in the XSL.

Step 14. Click Save Mapping Rule. The new mapping file will be displayed in the Mapping Rules list view page.

Step 15. To add more Content mapping rules, repeat Step 3 through Step 13. If you want to view the mapping file, go to the Content Mapping Rules page where you will see the records listed in the table. Click on the selected mapping rule’s hyperlink under Name in the table. If you wish to update the selected mapping file, click Edit Mapping Rule.

Step 16. To edit the Content mapping rule, select the mapping rule and then click on the corresponding Edit button . The screen appears. Update the entries of the mapping rule as necessary. Click Save Mapping Rule.

Step 17. To delete the Content mapping rule, tick the Select checkbox of the mapping rule you want to delete. As you see a mark appear in that checkbox, click Delete Mapping Rules. The message box, “Are you sure you wish to delete these Mapping Rules?” will appear. To confirm the deletion, click OK. The selected mapping file will be removed from the Mapping Rules page and the Content Mapping Rules page.
