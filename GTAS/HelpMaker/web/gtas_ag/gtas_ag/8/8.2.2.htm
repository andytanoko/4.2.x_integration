8.2.2 To create a new conversion rule mapping file

Conversion rules are used to convert data files from one format to another. Currently GridTalk supports the file formats for conversion such as Excel, XML, Positional Text and Delimited Text. It can also extract/write data from/to database. Conversion Rule can be created using a specialized Conversion Rule editor program (GridMapper TM ) or any text-based program. Once created, Conversion Rule can be inserted into GridTalk.

Step 1. Click Mapping. Then click Conversion Rule under Mapping Files. The Conversion Rule Mapping Files list view page is displayed.

Step 2. Click Create Mapping File. The Mapping File detail view page is displayed.

Step 3. In the Mapping File detail view page, enter the mandatory fields.

Step 4. Select Conversion Rule for the Type field.

Step 5. Select the file by clicking Browse. The File selector is displayed to select the Conversion Rule file.

Step 6. Click Save Mapping File. The new mapping file will be displayed in the Mapping Files and Conversion Rule Mapping Files pages.

Step 7. To add more Conversion Rule mapping files, repeat Step 2 through Step 6. If you want to view the mapping file, go to the Conversion Rule Mapping Files page where you will see the records listed in the table. Click on the selected mapping file’s hyperlink under Name in the table. If you wish to update the selected mapping file, click Edit Mapping File.

Step 8. To edit the Conversion Rule mapping file, select the mapping file and then click on the corresponding Edit button . The Mapping File page is displayed. Update the entries of the mapping file as necessary. Click Save Mapping File.

Step 9. To delete the Conversion Rule mapping file, tick the Select checkbox of the mapping file you want to delete. As you see a mark appear in that checkbox, click Delete Mapping Files. Then you will be prompted to confirm the message box, “Are you sure you wish to delete these Mapping Files?” Click OK. The selected mapping file will be removed from the Mapping Files page and Conversion Rule Mapping Files page.

To create a Conversion Rule mapping file that uses external libraries:

If your conversion file uses external libraries, you need to perform the following steps in addition to those described above:

Step 1. To find out what libraries your file uses, open the conversion file in an editor and search for the tag “<Dependencies>” under “<RulesContainer>”. All the *.class and *.jar files listed within “<Dependencies>” are your external libraries. This step is optional if you already know the external libraries that are required.

Step 2. Copy all these library files into GridTalk’s external library folder, which by default is <GRIDTALK_HOME>/<jboss>/server/default/lib. This path is specified in <GRIDTALK_HOME>/data/GNapps/conf/default/pdip/base/xml/convertor.config
