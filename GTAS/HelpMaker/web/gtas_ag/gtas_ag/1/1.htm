1 About the Guide

This GridTalk 4.2 Administrator guide is especially designed for the users endowed with the administrator’s rights. Aided by the screens, the guide contains the step-by-step instructions to enable you to perform the administrative tasks while you are getting acquainted with the features in GridTalk 4.2.

It covers mainly the following features, which you should be able to understand completely:

· Administration

Easy-to-use instructions on how to create users, establish the access rights for users, assign roles to users, and lastly create alerts and message templates.

· Communication

A quick summary of the available profiles necessary to get GridTalk connected so as to perform the information exchange with trading partners smoothly and efficiently. It includes information on Security, Communication, Channel and Package, and how to establish certificates for owners and partners.

· Mapping

A comprehensive overview of how the available mappings are used to configure a partner function, triggering a business process between trading partners. Mappings involve the creation of document type, file type, mapping rules and certificate mappings.

· Process

A more comprehensive overview of partner functions, including RFC, Ports and Triggers to be configured. Also includes an introduction to RosettaNet in which you are expected to configure a single-action or double-action process definition to send (initiator, requestor) and receive (responder) the RosettaNet-based documents and to view and cancel process instances of the process documents.

· Export and Import Configuration

A quick overview on how GridTalk can export and import available configuration data using XML. Such configuration data include alerts, alert actions, alert triggers, message template, document type, file type and so on.
