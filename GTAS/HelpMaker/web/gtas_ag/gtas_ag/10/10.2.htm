10.2 Configuring Backend Receiver

The receiver.jar file is a simplified backend listener for GridTalk to export files to the backend using a RFC.

The command line options for the receiver.jar are as follows:

Where each parameter represents as follows:

<listen_port>

Listener Module Listening Port, mandatory, must tally with RFC configuration. 6767 is conventionally used.

<gt_ip>

GridTalk IP Address, mandatory

<gt_listen_port>

GridTalk Backend Listening Port, mandatory, GridTalk default is 6763

<smtp_host>

The SMTP server for sending error email

<email_on_error>

Notification Email Address, optional

T<concur_proc_no>

The number of concurrent processing allowed, e.g. T20. Optional. Defaults to T10 when not specified.
