10.1.5 Senderlite.jar

The senderlite.jar file is a simplified backend sender for GridTalk. It accepts all the parameters on the command line, and there are no directories or configuration files to set up. It also does not litter the directory with the output log files. The command line options for the senderlite.jar are as follows:

where

-I <gt_listen_ip>

IP address of the Grid Talk server , e.g. 127.0.0.1 for local GridTalk server

-O <gt_listen_port>

Listening port on the Grid Talk server, e.g. 6763 for default

-N <gt_user>

GridTalk user making the connection to GridTalk server, e.g. admin

-W <gt_user_pass>

Password of the GridTalk user, use the Tool “maskedPass.bat” under <GRIDTALK_HOME>/bin/admin to generate the masked password.

-S <smtp_host>

SMTP server to use to send to error notifications (optional)

-E <email_on_error>

E-mail to receive error messages (optional)

-D <doc_type>

Document type of the import file (optional)

-P <partner_id>

Partner ID of the partner to receive the files (optional)

-F <files>

Full path of the file to be imported to GridTalk. To specify more than one file, use ; to separate them.

-A <attachments>

Full path of the file to be attached to the import file. To specify more than one, use ; to separate them (optional)

-X

Delete the import files after sending to GridTalk (optional)

-Y

Delete the attachment files after sending to GridTalk (optional)

-U <uid>

Unique identifier for the document to be imported (optional). Used for linking imported files with backend system identifiers.

-PS <procid>

Process Instance Id of the request message. Used for linking response message to request message of the same process instance.

-Z

Switch on debugging messages (optional)

Take note that it is not required for the above parameters to appear in any particular order.

This program allows a simpler backend configuration rather than setting up many configuration files that the sender.jar file requires as earlier described in the sender.ini/directory.file/info files. Both the sender.jar and senderlite.jar files perform the same function though. You can choose to set up the configuration files or use senderlite.jar and modify everything in one batch file. If you intend to dump separate files into many directories and want to send all of them at once, the sender.jar file is appropriate. However, in the case of simple imports, use the senderlite.jar file instead.
