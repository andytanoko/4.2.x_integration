10.1.2 Sender.ini

The sender.ini file is an XML file that is used in the backend sender for importing the document into GridTalk. It specifies the communication properties with GridTalk, the email settings for error case reporting, and the default location to pick up the documents to be imported.

The following shows a sample of a sender.ini:

Property key

Usage Description

PORT_NAME

An arbitrary name for this configuration. Mandatory.

GT_IP

IP address or host name of the GridTalk to send the files to. Mandatory.

GT_PORT_NUMBER

GridTalk backend listening port, default configured in GridTalk is 6763. Mandatory.

USER_NAME

User name required to login to GridTalk during import. The user name must be a valid user defined in GridTalk. Mandatory.

PASSWORD

User password required to login to GridTalk. Mandatory, use the Tool “maskedPass.bat” under <GRIDTALK_HOME>/bin/admin to generate the masked password.

BIZ_ENT_ID

The Business Entity ID of the user. Mandatory.

SMTP

SMTP server used to send error emails.

EMAIL

Email address of the person to receive the error emails. Optional.

HOST_DIR

Directory for picking up the documents to send. Applicable only if directory.file is not specified in command line option.

ATTACHMENT_DIR

Directory for picking up the attachments to send with the documents, if any. Applicable only if directory.file is not specified in command line option.

LOCKFILE

Name of the file that will be created to lock the directory while import is in progress. Mandatory.

TIMEOUT

Time limit (in minutes) to prevent staled lock situation by a sender process that has been terminated abnormally. Mandatory.

COMMAND_FILE

Specify an external program/script to be executed before sending the files into GridTalk, the string ";%FILENAME"; represents the filename of the file to be imported. If there are multiple files in the HOST_DIR (or Directory in directory.file), each of the files will be processed by this external program. Optional.
