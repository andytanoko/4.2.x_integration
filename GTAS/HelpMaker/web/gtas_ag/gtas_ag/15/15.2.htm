15.2 Importing Configuration Data

Step 1. Click Administration tab. This will bring you to the Administration page.

Step 2. Click Import Configuration Records. Then the Import Configuration Records page is displayed to prompt you to select the configuration file to import into GridTalk.

Step 3. Click Browse to select the zip file from which to import the configurations. The File Selector dialog box appears. Once you have selected the file, click OK.

Step 4. Next, you are given two choices whether to overwrite conflicting records manually or automatically. If you choose Automatically, you will not be prompted to select the conflicting records to be overwritten, all conflicting records will be overwritten by the records in the zip file. However, you are still advised to choose to manually overwrite those records.

Step 5. Click Import Records. If you choose Manually and there are conflicting records go to Step 6 otherwise the configurations will be imported and you will return to the Administration page.

Step 6. If you wish to select and display all these conflicting records fully, just tick the empty checkbox next to the Conflicting Records. Each parent record is accompanied by one forward-arrow button, which is used to expand all its child records. When this button is toggled to become a down-arrow one, it will collapse its child records. Tick the records that you want to overwrite.

Take note that Partner configuration records, when imported and overwriting the existing conflicting records, will default to Disabled state, except for those that were in the Deleted state when they were exported, which still remain as Deleted.

Step 7. Once done, click Import Records again. Wait for a few moments until the import process is successfully completed. Otherwise you will encounter the error message, that is, GridTalk error.

Step 8. To import more configuration records, repeat Step 2 through Step 7.
