15.1 Exporting Configuration Data

Step 1. Click Administration tab. This will bring you to the Administration page.

Step 2. Click Export Configuration Records. Then the Export Configuration Records page is displayed with an existing list of all available system and user-defined configurations each containing a blank checkbox to export into your system.

Step 3. If you wish to select all configurations to be fully displayed in a parent-child structure, just tick the empty checkbox next to the Exportable Records. Each configuration is accompanied by one forward-arrow button, which is used to expand all its records. When this button is toggled to become a down-arrow one, it will collapse its records.

Step 4. Select the required configuration(s) to export. For example, Process Definition. The Process Definition folder will expand to list the records that you have earlier created.

Step 5. Once done, click Create Export File. The system will generate the zip file for that configuration data you have selected. It will be shown and click-able at the Configuration File field. The filename is defaulted to be numerical but you can rename it to give a suitable name of the configuration zip file.

Step 6. Click it and the File Download dialog box appears asking you to download it into the temporary folder, and then to rename and save it to your own folder directory.

The zip file you have saved contains the configuration records that you have selected to export, as well as the configuration records which they depend on. For example: the Process Definition records depend on DTD, Dictionary, and Schema Mapping Files. Those mapping files that the process definitions depend on will also be exported into the zip file. Take note that if the exported configuration record has a physical file component, the physical file will also be exported into the zip file. The exported configuration records are in XML format.

Step 7. To export more configuration data, repeat Step 2 through Step 6.
