9.1 Configuring Partner Functions

Now that you know a partner is someone you need to build a specific function to trigger the business process, each partner may have specific functions, which are called partner functions.

Partner functions are used to define the workflows in the exchange of business documents.

A Partner Function is composed of one or more workflows which are made up of workflow activities. For instance, on receive of a document you can generate an automated response to send back to your partner, and at the same time, you can export it for your backend ERP system.

The table illustrates the list of workflow activities and their meanings:

Activity Type

Description

Exit to export

This exit allows you to configure the workflow action, which the documents can flow into the Export folder.

Exit to outbound

This exit allows you to configure the workflow action, which the documents can flow into the Outbound folder.

Exit to Channel

This workflow action is configured for the user to send the documents to the trading partner based on the recipient partner Id field in the GridDocument. This activity can only be configured in the Outbound folder.

Mapping Rule

This mapping rule transforms, converts or splits a user document.

Exit to import

This exit allows you to configure the workflow action, which the documents can flow into the Import folder.

Raise alert

This workflow action is configured to raise an alert when the event takes place.

Exit workflow

This exit signals the end of the chain of workflow activities in the current folder. The immediate workflow activity that follows, if any, may act upon the document in the originating folder.

Save to Folder

This workflow action is configured to allow the documents to save into the current folder in its current state.

User Procedure

The user procedure makes use of configured external codes/programs to execute configured parameters to pass to external codes/program and expected return value. The user also can configure actions to raise alerts based on their return value. For example, user procedure to perform FTP.

Exit to Port

This exit allows you to configure the workflow action, which the documents can be exported through the indicated port. This activity can only be configured in the Export folder.

For the multiple workflow example mentioned above, the partner function can be defined with the following sequence of workflow activity types:

  1. Exit to Outbound
  2. Mapping Rule – Transform to response
  3. Exit to Channel
  4. Exit Workflow
  5. Exit to Export
  6. Mapping Rule – Convert to Flat File
  7. Exit to Port – ERP Backend
  8. Exit Workflow

As you may notice, activities 1 to 4 constitute one workflow, and activities 5 to 8 constitute another workflow. Each workflow in the Partner Function is demarcated by an Exit Workflow activity type.
