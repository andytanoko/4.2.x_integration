9.2 Configuring Triggers

In GridTalk, a trigger is either one condition or a series of conditions that sets the partner function into action. A trigger can be created based on the type of level, which a particular partner function requires in order to process the document workflow. For example, a partner function that only needs a single trigger (e.g., the document being a purchase order) to be activated is considered a Level 1 partner function. A partner function that requires multiple triggering (e.g., the document being a purchase order sent by an overseas customer) is ranked higher.

The five available levels of trigger are explained:

  • Level 0 trigger: No conditions. Only system partner functions can be ranked 0. It is non-editable.
  • Level 1 trigger: Document Type.
  • Level 2 trigger: Document Type, Partner Type.
  • Level 3 trigger: Document Type, Partner Type, Partner Group.
  • Level 4 trigger: Document Type, Partner Type, Partner ID
