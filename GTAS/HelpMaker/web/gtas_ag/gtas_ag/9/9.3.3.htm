9.3.3 To configure a request or response action

Both the request and response configurations share the same parameters.

Step 1. Click Edit Request Action Configurations. This will bring you to the page where you need to complete the Files, Details, Options and Encryption panes.

Step 2. First go to the Files pane and select the following inputs:

Message Type

This field represents the message DTD for the request message.

Diction File

This field provides dictionary values for validating service content

XML Schema

XML Schema can be used to perform extra validation like field length, etc

Step 3. Next go to Details pane and complete the following fields:

Business Activity Identifier

From Service Header. Use Activity Name in PIP Specification’s Business Activity Descriptions table.

Business Action Code

From ‘Business Action in FSV’ column of the PIP specification’s ‘Business Action – Business Document Mapping’ table.

Retries

This field is the number of times to retry before the process fails.

Time to Acknowledge (in secs)

This field is the timeout to receive the request acknowledgement. The timeout is specified in seconds.

Step 4. Proceed to the Option pane and complete the following fields:

Is Authorization Required

This checkbox specifies if the partner requires explicit authorization to initiate the process.

Is Non-Repudiation Required

This checkbox specifies if the non-repudiation is required .

Is Secure Transport Required

This checkbox specifies if the secure transport is required. GridTalk provides the secure transport by default so this option only affects HTTP transport.

Disable DTD

If set, the DTD validation will be disabled

Disable Schema

If ticked, XML Schema validation will be disabled.

Validate At Sender

If ticked, the outgoing PIP message contents will be validated.

Step 5. Proceed to the Encryption pane and complete the following fields:

Disable Encryption

If ticked, the message will not be encrypted.

Disable Signature

If ticked, the message will not be signed.

Digest Algorithm

This field gives the two options such as MD5 or SHA1 to decide which algorithm to use for generating the RosettaNet/CIDX message digest (used with signatures and in the acknowledge messages).

Encryption Algorithm

This field gives the choices of RC2 or 3DES to specify the method of encryption.

Encryption Algorithm Length

This field indicates how difficult it will be for the unauthorized users to crack the information. Choosing an encryption key length usually depends on how sensitive or valuable the information is that you're protecting. The larger key length is used to ensure that there is added protection to the information. 3DES supports 168-bit keys while RC2 supports 40-bit, 64-bit, and 128-bit keys.

Step 6. Lastly if compression is required, go to Compression pane and complete the following fields:

Is Compression Required

If ticked, the message contents will be compressed. The compressed algorithm used is ZLIB.

Step 7. Click Save Process Action.
