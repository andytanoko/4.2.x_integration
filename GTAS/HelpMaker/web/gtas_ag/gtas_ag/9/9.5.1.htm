9.5.1 To create a new process mapping

Step 1. Click Process. The browser brings you to the list of submenu under Process in the left pane page.

Step 2. Click Process Mappings. The Business Process Mappings list view page is displayed.

Step 3. To create a new process mapping, click Create Process Mapping. The Business Process Mapping detail view page is displayed.

Step 4. Select the process definition for the Process to Map field. If the process definition is not listed, click on the icon to create a new process definition.

Step 5. Select Partner. If the partner is not listed, click on the icon to create a new partner.

Step 6. Specify the role, which you are participating in. Initiating Role means you act as an initiator. Responding Role means you are a responder.

Step 7. Next proceed to the Channel dropdown list to select the channel. The list contains all the available channels assigned to that partner’s business entity. When you are engaged in a process of the particular process definition with the partner, the selected channel will be used to send all documents of that process to that partner.

Step 8. Next select the document type from the dropdown list. This list contains all document types defined in the application. When you are engaged in a process of the particular process definition with the partner, all your received business documents will be assigned the selected document type, which is usually used to invoke the appropriate trigger.

Note also that if the selected Process to Map is a single-action PIP, and the Initiating Role is selected for Participating In field, then Document Type will not be shown for selection as there is no need to.

Step 9. To save the process mapping, click Save Process Mapping.

Step 10. To edit the process mapping, click on the Edit button next to the process mapping you want to select . The screen appears. Then update the relevant entries. Always save whatever you have updated.
