9.3 Configuring Process Definitions

GridTalk provides with supply chain partners the capacity to send and receive RosettaNet messages according to the specification described in the RosettaNet Implementation Framework (RNIF) .

The RosettaNet Implementation Framework Specification is an implementation guideline for applications that implement RosettaNet Partner Interface Processes (PIPs).

Each PIP contains a technical specification based on the RosettaNet Implementation Framework (RNIF), a Message Guideline document with a PIP-specific version of the Business Dictionary and an XML Message Guideline document (an XML document type definition or DTD).";

A Process Definition defines a business transaction according to a PIP. It specifies the content of the messages, the choreography and the performance controls for the transaction. Each Process Definition can be created as a Single-action process or a Two-action process.

A Single Action PIP expects only an acknowledgment to a Request message. If an acknowledgement is not received within the timeout, the message will be resent.


A Double Action PIP requires a response to a request. The responder has to acknowledge the initial request and sends a response back to the Initiator. The Initiator will acknowledge the response message.
