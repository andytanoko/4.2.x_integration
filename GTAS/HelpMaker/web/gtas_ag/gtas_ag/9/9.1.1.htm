9.1.1 To create a new partner function

Step 1. Click Process.

Step 2. Click Partner Functions. The Partner Functions list view page is displayed. The table list shows the partner function ID, its purpose and how it is triggered.

Step 3. To create a new partner function, click Create Partner Function. The screen appears.

Step 4. Enter Id. This field identifies the partner function you are creating.

Step 5. Enter Description. This field gives the descriptive purpose of that partner function.

Step 6. Select Trigger On. This field allows you to select when that partner function can be triggered. The available methods of triggering a particular partner function are Import, Manual Export, Manual Send, and Receive.

Import – this partner function can only be triggered on import of a document.

Manual Export – this partner function can only be triggered on manual export of a document.

Manual Send – this partner function can only be triggered on a manual send of a document.

Receive – this partner function can only be triggered on receiving a document.

Step 7. Next go to the Workflow Activities section. To create a workflow activity, click Create Workflow Activity. The screen appears. Enter the following fields for the General section.

Description:

This field gives the descriptive purpose of that partner function. This field is not enabled for entry.

Activity Type:

This field allows you to configure one or more workflow activities in the partner function.

Step 8. Next proceed to the Suspend section to enter the following fields:

Dispatch Interval (sec):

This field allows you to specify the interval after which you can resume the suspended workflow activities.

Dispatch Count:

This field allows you to specify the number of workflow activities to resume at a time.

The ‘Suspend’ options allow workflow processing to be suspended for a configurable amount of time before continuation. This feature is useful under situations of heavy load e.g. large amount of concurrent incoming documents and there's need to regulate the concurrent execution of workflow processes by limiting the number of concurrent execution of the same workflow activity at any instant.

An example of usage is to suspend the Export of incoming RosettaNet documents till a later dispatch time, giving resources for processing the return of the RosettaNet Receipt Acknowledgements to the sender. In this case, in the ";Exit to Export"; (or Exit to Port) activity, a Dispatch Interval can be configured to force the processing to proceed several seconds later. This relinquishes resources to another workflow process, which is started for sending RosettaNet Receipt Acknowledgements. The Dispatch Count controls the maximum number of suspended ";Exit to Export"; activities that will be dispatched at each dispatch time.

Step 9. Click Save Workflow Activity(s) to save the Workflow Activity.

Step 10. To add more workflow activities, repeat Step 7 through Step 9. To modify a Workflow Activity, click the Edit button against the Workflow Activity you want to change. To delete a Workflow Activity, tick the checkbox for the Workflow Activity and click Delete Workflow Activity.

Step 11. To change the order of the workflow activities (this affects the sequence of execution of the workflow), tick the checkbox for the workflow activities to change order, and click the Up or Down accordingly.

Step 12. Click Save Partner Function. The newly created partner function will be displayed in the Partner Functions view list page.

Step 13. To edit the existing or current partner function, click on the Edit button next to the partner function you have selected. Update the entries of the partner function as necessary. Click Save Partner Function.

Step 14. To delete the existing or current partner function, select the check box of the partner function you want to delete. Click Delete Partner Functions. The selected partner function will be removed from the Partner Functions list.
