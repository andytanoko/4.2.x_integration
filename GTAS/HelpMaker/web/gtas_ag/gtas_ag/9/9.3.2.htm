9.3.2 To create a double action process definition

The double-action process performs both the Request and the Response. The Request allows you to configure the initial Request message. The Response allows you to configure the Response message.

Step 1. Click Process.

Step 2. Click Process Definition. This will bring you to the page where you need to complete the General, From and To panes.

Step 3. First go to the General pane and enter the Name for the new process definition.

Step 4. If the process is a synchronous PIP, tick the Is Synchronous checkbox. Otherwise, leave it as unchecked.

Step 5. Select Double Action Process for the Process Type field.

Step 6. Click Edit Request Action Configurations to configure the request action settings. Click Edit Response Action Configurations to configure the response action settings. Refer to “To configure a request or response action” for more details.

Step 7. In the General pane, enter the rest of the following fields:

Action Timeout

This field is the time for the recipient to complete the action and send the Response document

RNIF Version

This field provides RNIF1.1 or RNIF2.0 to select. This version affects the packaging and transport of the message. RNIF2.0 compiles to RosettaNet Implementation Framework: Core Specifications version V02.00. RNIF1.1 implements RosettaNet Implementation Framework Specification version 1.1 dated 8 Nov 1999.

PIP Process Code

This field represents the PIP alphanumeric designation (e.g ‘3C3’)

PIP Version

This field specifies the version of the PIP. The value must be obtained from the PIP Version Identifier element in the corresponding PIP specification.

Usage Code

This field gives Test or Production to choose depending on use.

Step 8. Next go to the From pane to complete the following fields.

Partner Class Code

This field represents the partner classifications, which are described in the Global Partner Classification Code element of the Service Header Part Message Guideline of the RNIF1.1 specifications. For RNIF2.0 messages, this field is not used.

Partner Role Class Code

From Role Name column of the PIP specification’s Partner Role Descriptions table

Business Service Code

From Network Component in FSV column of the PIP specification’s Network Component Specification table

Step 9. Proceed to the To panel to complete the following fields.

Partner Class Code

This field represents the partner classifications, which are described in the Global Partner Classification Code element of the Service Header Part Message Guideline of the RNIF1.1 specifications. For RNIF2.0 messages, this field is not used.

Partner Role Class Code

To Role Name column of the PIP specification’s Partner Role Descriptions table

Business Service Code

To Network Component in FSV column of the PIP specification’s Network Component Specification table

Step 10. Lastly go to the Document Identifier panel to complete the following fields.

Request Document Identifier

This field represents the XPath to the unique document ID of the request document

For example,

//Pip3A4PurchaseOrderRequest/thisDocumentIdentifier/ ProprietaryDocumentIdentifier

Response Document Identifier

This field represents the XPath to the unique document ID of the response document

For example,

//Pip3A4PurchaseOrderConfirmation/thisDocumentIdentifier ProprietaryDocumentIdentifier

Response Document Request Document Identifier

This field represents the XPath, in the response document, to the document ID that refers to the request document, to which the response document is responding.

For example,

//Pip3A4PurchaseOrderConfirmation/requestingDocumentIdentifier/ProprietaryDocumentIdentifier

User Tracking Identifier

This optional field represents the XPath to an element whose value the user selects to identify the process instance. Unlike the process instance ID, which uniquely identifies a process instance, the ID extracted according to this XPath may not be unique. If carefully chosen, however, it helps the user track process instances by providing a user-friendlier identifier.

For example,

//Pip3A4PurchaseOrderRequest/thisDocumentIdentifier/ProprietaryDocumentIdentifier

See Chapter 9.4.1 for more explanation on the purpose of using this field.
