12 Internal Scheduler

The GridTalk Internal Scheduler function allows the user to schedule jobs internally in GridTalk to run at specific times once, minutely, hourly, daily, weekly, or monthly. The function is accessible in Admin mode under the Administration tab using the Scheduled Task list view page.

There are three types of schedule tasks: one which is system task, the second one is user procedure task and the third one is the DB Archive task. You are only allowed to schedule new user procedure tasks and DB Archive tasks. System tasks are pre-configured with the GridTalk application but you are allowed to modify the schedule timings. So far, there are two pre-scheduled system tasks, and one pre-scheduled user procedure task:

  • Housekeeping – this helps you to housekeep the temporary files and log files generated from the day-to-day transactions of the GridTalk application. This task is disabled by default. See Chapter 13.4 Scheduling housekeeping jobs for more details.

  • License Checking – this helps you to monitor the validity period of your GridTalk license and will notify you by email before and after the license expiry so that you can promptly take actions to renew the license with GridNode. This task is enabled by default and is scheduled to run daily at midnight. You may modify the schedule timing as well as the frequency.

  • Certificate Expiry Checking – this helps you to monitor the validity period of all your certificates as well as partner certificates, and will notify you by email before and after any certificate expiry so that you can promptly take actions to renew the certificate. This task is enabled by default and is scheduled to run daily at the time of installation. You may modify the schedule timing as well as the frequency. This is a user procedure task, which means the way the expiry checking is performed may be changed by replacing the user procedure or modifying the user procedure.
