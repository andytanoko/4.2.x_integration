5.1.1 To create a new user

Step 1. Click Users under the Administration menu in the left pane of the browser. The Users list view page is displayed where you will see two default users, namely, “admin” and “guest”.

Step 2. In the Users list view page, click Create User. Take note that prior to creating new user, its business entity must first be created.

Step 3. In the Account Details tab page, enter the necessary fields. Some fields that are highlighted blue are mandatory. For example, User ID and Name.

Step 4. Go to Roles. Select the role from the left listbox to be added to the right listbox. Click on the right pointing arrow to add the role. Click on the left pointing arrow to remove the role from the listbox.

Step 5. Next go to Business Entities. Select the business entity from the left listbox to be added to the right listbox. Click on the right pointing arrow to add the business entity. Click on the left pointing arrow to remove the business entity from the listbox. But if the business entity you want to select is not found in the left listbox, click on the icon to create a business entity. (Refer to Chapter 4.2 for creating a business entity in GridTalk 4.2 User Guide).

Step 6. Next click on the Password tab. The screen appears as follows:

Step 7. Enter the password to be used by the new user in the Enter Password and Confirm Password fields. The password must be entered twice in order to avoid the inconvenience that would result in a typing error made when entering the password which being hidden by the ‘*’ displayed would go unnoticed otherwise

Step 8. Click Save user details. The new user will be displayed in the list table of the Users page.

Step 9. To add more new users, repeat Step 2 through Step 8. If you want to view the complete profile of the selected user, go to the Users page where you will see the user records listed in the table. Click on the selected user’s hyperlink under User Id in the table. If you wish to update the selected user, click Edit User.

Step 10. To edit the existing user, select the user you want to edit. Click on the Edit button next to theuser you have just selected. By default, the Account Details tab page is displayed.

Step 11. Click on the Account State tab. The browser brings you to the page with the following information:

State:

Select Enabled or Disabled. If enabled, the owner is able to log on to GridTalk. But if disabled, the owner is unable to log on to GridTalk regardless of which password they enter – be it correct or incorrect. It is the responsibility of the administrator who has the appropriate access rights to disable or enable user accounts.

Last Login:

Displays the date and time when the user last logged on to GridTalk.

Last Logout:

Displays the date and time when the user last logged out of GridTalk.

Frozen:

Displays Yes or No to indicate that the user account is frozen or not frozen respectively. For example, No means that the account is not frozen.

Failed Logins:

Indicates the number of failed attempts that the user has made trying to log on to GridTalk. This will be reset every time after a successful login.

Creation Time:

Displays the date and time that the user account was created.

Created by:

Displays the person who created the user account. This person will be someone responsible for managing user accounts.

Step 13. In the Edit User Details list, update the entries for the user as necessary. Click Save user details. Otherwise click Cancel to abort the changes.

Step 14. To delete the existing user, tick the Select check box of the user you want to delete. Take note that you can delete as many users as you wish using multiple selections at a time. Once you see a mark appear in each checkbox of the highlighted users, click Delete Users. The selected user(s) will be removed from the list table of the User page. Take note that the User ID of the deleted users cannot be re-used.
